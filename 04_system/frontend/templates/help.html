快速入门
1
配置 API 密钥

前往左侧导航栏的"设置"页面，输入您的 API 密钥。系统支持 OpenAI、Gemini、Claude 等多种主流 LLM 服务。

2
输入问题描述

在"代码生成"页面，详细描述您想要解决的编程问题。描述越清晰、边界条件越明确，生成的代码越准确。

3
编写测试用例

使用 Python 原生的 assert 语句编写测试用例，TraceCoder 将以此为基准，自动进行插桩测试和代码修复。

4
一键开始生成

点击"开始生成"按钮，多智能体系统将自动为您生成代码、沙箱执行测试，并在测试未通过时启动自我修复循环。

调试循环详解 (Debug Loop)

当初始代码未能通过预设的测试用例时，TraceCoder 核心机制将介入：

  1. 代码插桩：自动为代码添加诊断探针，收集运行时变量信息。
  2. 执行追踪：沙箱运行插桩后的代码，抓取完整的执行轨迹 (Trace)。
  3. 智能分析：分析智能体根据轨迹数据与断言错误，精准定位 Bug 根源。
  4. 代码修复：修复智能体参考历史经验，输出修正后的代码版本并重新验证。
核心技术机制
短期记忆机制 (Short-Term Memory)

整合历史经验学习与安全回滚机制，提供智能的修复过程管理

修复历史记录
  • 实时记录每一次修复尝试与结果
  • 避免模型陷入重复相同错误的死循环
  • 为后续修复提供动态历史参考
智能回滚保护
  • 实时跟踪最佳测试通过率
  • 检测到性能退化时自动介入
  • 防止无效修改，回滚至最佳版本
使用指南
设置页面详解
  1. API 核心配置：输入您的 API 密钥和基础 URL，点击"获取模型列表"按钮加载可用模型
  2. 模型选择：从加载的模型列表中选择您想要使用的模型，支持多选
  3. 运行参数：
    • 最大调试次数：设置代码失败时的最大修复尝试次数（推荐 3-5 次）
    • 执行超时：单次代码执行的最大时间限制（单位：秒）
    • 回滚阈值：连续无改进多少次后触发回滚机制
    • 代码插桩：启用后会自动在代码中添加诊断探针，帮助定位问题
  4. 配置完成后点击"保存配置"按钮，设置会自动保存到本地
代码生成页面使用
  1. 问题描述：
    • 清晰、准确地描述您想要解决的编程问题
    • 包括输入输出格式、边界条件、示例等信息
    • 描述越详细，生成的代码质量越高
  2. 测试用例：
    • 使用 Python 的 assert 语句编写测试用例
    • 每个 assert 占一行
    • 覆盖正常情况、边界情况和异常情况
    • 示例：assert fib(0) == 0、assert fib(1) == 1
  3. 快速选择示例：点击预置的示例按钮，快速填充问题描述和测试用例
  4. 查看配置：页面顶部显示当前已配置的参数，点击可跳转到设置页面修改
  5. 开始生成：
    • 点击"开始生成"按钮启动代码生成流程
    • 可以点击"暂停"按钮暂停，"继续"按钮恢复
    • 点击"终止"按钮立即停止整个流程
执行进度查看
  1. 紧凑模式：默认显示横向进度条，节省空间
  2. 展开详情：点击右侧箭头图标展开详细的竖向步骤
  3. 进度步骤：
    • 问题理解：解析您的问题描述
    • 初始生成：调用 LLM 生成第一版代码
    • 代码测试：在沙箱中运行测试用例
    • 结果分析：分析测试结果和错误信息
    • 调试迭代：多轮调试修复代码
    • 任务完成：输出最终通过所有测试的代码
  4. 智能体状态：查看 HLLM 和 RM 机制的实时运行状态
代码输出与保存
  1. 实时流式输出：代码会逐字逐句实时显示，类似打字机效果
  2. 代码高亮：自动对代码进行语法高亮，方便阅读
  3. 行数统计：右上角显示当前代码的总行数
  4. 操作按钮：
    • 复制代码：一键复制代码到剪贴板
    • 下载文件：将代码保存为 solution.py 文件
历史记录与示例
  1. 历史记录：
    • 查看所有之前生成的代码任务
    • 可以查看详细信息、重新运行或删除历史记录
    • 首页会显示最近的 5 条任务记录
  2. 示例库：
    • 预置了丰富的编程问题示例
    • 涵盖算法、数据结构、实用工具等多种类型
    • 点击示例可以直接跳转到代码生成页面并自动填充内容
常见问题
如何获取 API 密钥？

您可以从以下官方平台注册并获取 API 密钥：

  • OpenAI: platform.openai.com
  • Google AI: ai.google.dev
  • Anthropic: console.anthropic.com
支持哪些编程语言？

目前 TraceCoder 核心引擎主要支持 Python 的动态代码插桩、执行和修复验证。未来版本计划增加对 JavaScript 和 Java 的支持。

测试用例如何编写？

使用标准的 Python assert 断言语句编写，每行定义一个独立用例：

assert func(arg1) == res
assert func(0) == 1
assert func(-1) == 0
代码执行安全吗？

代码默认在后端受限沙箱环境中执行，并设置了严格的运行时长和内存限制。但仍建议您不要让 AI 生成涉及本地系统文件操作或高危网络请求的代码。

生成失败的原因可能是什么？
  • API 密钥无效、格式错误或额度耗尽
  • 本地到 API 提供商的网络连接受阻
  • 问题描述存在严重逻辑歧义
  • 提供的测试用例本身互相矛盾
