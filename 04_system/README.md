# 第 7 章：运行时轨迹驱动的多智能体代码生成与自动调试系统

## 系统定位

本目录归档论文第 7 章系统实现相关材料。系统面向多智能体代码生成与自动调试场景，将跨模型协同、运行轨迹采集、智能体分析、补丁生成、测试验证、回滚和结果展示集成为可演示、可检查、可部署的工程原型。

## 功能模块

| 模块 | 功能说明 | 归档位置 |
|---|---|---|
| 任务管理 | 创建代码生成或修复任务，管理任务状态和历史记录 | `frontend/`、`backend/` |
| 代码生成 | 根据任务描述、上下文和模型配置生成初始代码 | `backend/` |
| 运行轨迹采集 | 执行插桩、测试运行和轨迹收集 | `backend/`、`test_records/` |
| 多智能体分析 | 调用插桩、分析、修复、仲裁等智能体协同工作 | `backend/`、`api_docs/` |
| 自动修复 | 生成补丁、执行测试、记录修复轮次和回滚 | `backend/`、`database/` |
| 结果展示 | 展示轨迹、错误原因、补丁差异、测试结果和最终状态 | `frontend/`、`screenshots/` |
| 系统管理 | 配置模型接口、任务队列、日志和部署参数 | `deployment/` |

## 前端归档要求
