## 项目简介

本目录用于集中归档 Ye Wenguang 硕士毕业论文及相关科研制品，覆盖论文正文、方法代码、数据集、实验记录、结果图表、系统实现、PPT 报告、评审与答辩材料、过程记录和后续交接说明。归档目标是保证论文研究过程可检查、实验结果可追溯、系统实现可复现、后续学生可继续补充和维护。

本项目研究主题聚焦多智能体协同驱动的代码生成与自修复。论文围绕大语言模型在代码任务中的协同生成、轨迹分析、测试驱动修复、仓库级上下文建模和运行时自动调试系统展开，强调方法设计、实验验证和系统落地之间的一致性。

## 学位论文五部分核心内容概述

1. 基于跨模型协同的代码生成方法：围绕不同代码大模型的能力差异，设计多模型候选生成、结果评估、协同选择与互补增强机制，用于提升函数级代码生成质量。
2. 基于程序运行轨迹的多智能体代码生成与修复方法：通过插桩智能体获取运行轨迹，结合分析智能体、修复智能体、历史经验学习和回滚机制，支持错误定位与迭代修复。
3. 基于过程对齐的测试驱动代码生成与修复方法：面向测试驱动场景，建立初始代码与测试生成、过程对齐循环、仲裁分析和最优解选择机制，提高代码生成与测试反馈之间的一致性。
4. 基于上下文感知与记忆增强的仓库级代码生成方法：面向仓库级任务，结合上下文感知初始化、结构化记忆和多轮修复循环，提升跨文件理解与补丁生成能力；第 6 章实验归档重点包括 RepoExec 和 EvoCodeBench。
5. 基于运行时轨迹的多智能体代码生成与自动调试系统：将上述方法工程化，形成支持前后端交互、任务管理、运行轨迹采集、自动修复和结果展示的系统原型。

## 当前目录结构说明

- `00_admin/`：归档清单、提交要求、缺失项、状态表、版本日志和学生通知。
- `01_thesis/`：论文 PDF、源文件、摘要、图表、评审意见、答辩版本和最终版本。
- `02_papers/`：论文相关投稿、发表、回复和手稿记录。

- `05_datasets/`：数据集原始数据、处理数据、样例、许可证、元数据和数据说明。
- `06_experiments/`：HumanEval、MBPP、HumanEvalPlus、BigCodeBench、ClassEval、RepoExec、EvoCodeBench、消融实验和成本分析。
- `07_results/`：汇总表、原始输出、最终图表、案例分析和结果说明。

- `10_docs/`：项目说明、复现指南、环境配置、目录指南、科研制品说明、归档规则、常见问题和交接指南。

## 后续维护说明

1. 学生提交材料后，应优先更新 `00_admin/missing_items_todo.md` 和 `00_admin/archive_status_table.md`。
2. 每次新增或移动材料后，运行 `python3 tools/generate_tree.py` 更新 `tree.txt`。
3. 每次阶段检查前，运行 `python3 tools/check_missing_items.py` 生成 `00_admin/generated_missing_report.txt`。
4. 大文件、敏感数据和受版权限制材料应先脱敏或登记来源，不应直接提交到公开仓库。
5. 所有新增文件建议使用清晰命名，并在相应 README 或说明文档中记录来源、用途和复现方式。
