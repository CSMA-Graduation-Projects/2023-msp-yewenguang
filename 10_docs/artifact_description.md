# 科研制品说明

## 一、说明目的

本归档项目保存的是硕士毕业论文及其支撑性科研制品，而不仅是论文 PDF。科研制品包括能够支撑论文结论、复现实验过程、解释系统实现和完成毕业检查的所有材料。

## 二、制品范围

### 论文制品

包括论文最终 PDF、论文源文件、摘要、图表源文件、答辩版本、最终归档版本和评审修改记录。这些材料用于确认论文内容、版本和正式归档结果。

### 代码制品

包括第 3 至第 6 章方法代码、共享工具、提示词、评测脚本、系统前后端代码和部署脚本。代码制品应能说明方法如何实现，而不仅是论文中的伪代码。

### 数据制品

包括原始数据、处理后数据、样例、元数据、许可证和数据说明。数据制品用于确认实验输入来源、可公开性和预处理过程。

### 实验制品

包括实验配置、运行命令、日志、原始输出、统计脚本、结果表、消融实验和成本分析。实验制品用于支撑论文表格、图和结论。

### 图表制品

包括论文最终图表、图表源文件、绘图脚本和源数据。图表制品应能说明图表如何从实验结果生成。

### 系统制品

包括系统前端、后端、数据库、接口文档、部署说明、截图、演示视频、用户手册和测试记录。系统制品用于支撑论文第 7 章系统实现与演示。

### 管理制品

包括归档清单、提交要求、缺失项、状态表、版本日志、学生通知、时间线、周报和会议记录。管理制品用于归档检查、过程追溯和后续交接。

## 三、与论文内容的对应关系

| 论文部分 | 对应制品 |
|---|---|
| 第 3 章跨模型协同代码生成 | 多模型配置、候选输出、评估脚本、结果表和图表 |
| 第 4 章轨迹驱动修复 | 插桩代码、轨迹日志、分析输出、补丁记录和回滚记录 |
| 第 5 章过程对齐修复 | 测试生成、对齐循环、仲裁记录、最优解选择和实验结果 |
| 第 6 章仓库级代码生成 | 仓库上下文、结构化记忆、RepoExec 与 EvoCodeBench 实验、补丁和日志 |
| 第 7 章系统实现 | 前后端代码、部署说明、接口文档、截图、视频和测试记录 |

## 四、归档质量要求

1. 材料应真实对应论文内容。
2. 文件命名应清楚，能判断版本、来源和用途。
3. 实验结果应保留原始输出和统计过程。
4. 代码应提供运行入口和环境说明。
5. 数据应说明来源、许可证和使用限制。
6. 系统材料应能支撑演示或复查。

## 五、后续使用方式

后续学生或导师可通过本归档理解研究问题、定位方法实现、复现实验、检查结果来源、扩展系统功能和准备后续论文工作。任何新增材料都应同步更新目录树、缺失项表和版本日志。
