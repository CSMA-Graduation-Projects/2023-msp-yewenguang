# 常见问题

## 1. 只提交论文 PDF 是否可以？

不可以。本归档需要保存论文、代码、数据、实验、图表、系统、报告和毕业流程材料。论文 PDF 只是其中一部分。

## 2. 代码还没整理好，可以先提交压缩包吗？

可以临时提交，但必须提供说明文件，写明压缩包内容、来源、运行方式和后续整理计划。最终建议解压并按章节目录归档。

## 3. 数据集太大，无法放入 Git，怎么办？

请提交样例、元数据、下载脚本、许可证和完整数据的外部存储说明。说明中应包含访问方式、校验值、负责人和是否可公开。

## 4. 模型 API Key 是否可以放入配置文件？

不可以。真实密钥不得提交。请使用环境变量或本地私有配置，并提交 `config.example.yaml` 作为样例。

## 5. 实验结果只保留论文表格够不够？

不够。需要保留原始输出、日志、统计脚本和汇总表。论文表格应能追溯到源数据。

## 6. 论文图表需要提交源文件吗？

需要。应提交绘图脚本、源数据或可编辑文件，并说明对应论文图号或表号。

## 7. 如果某个实验无法复现，怎么处理？

请在对应实验目录写明无法复现的原因，例如外部服务变化、数据缺失、模型版本不可用或依赖失效，并尽量提交历史日志和原始输出。

## 8. 系统演示视频可以只放链接吗？

可以，但需要在 `04_system/demo_video/` 中保存说明文件，写明视频链接、访问权限、录制日期、内容概述和备份状态。

## 9. 评审意见和修改说明放在哪里？

论文评审意见放入 `09_review_and_graduation_materials/thesis_review_comments/`，逐条修改说明放入 `09_review_and_graduation_materials/revision_explanations/`。

## 10. 周报和组会记录是否必须提交？

建议提交。它们用于过程追溯和后续交接，尤其有助于理解实验设计、方法调整和论文修改过程。

## 11. 目录中已有 `.gitkeep`，可以删除吗？

当目录中已有实际材料后可以删除对应 `.gitkeep`。如果目录仍为空，应保留 `.gitkeep` 以便 Git 追踪目录。

## 12. 新增目录是否需要更新 `tree.txt`？

需要。每次新增、删除或移动文件后，应运行 `python3 tools/generate_tree.py` 更新目录树。

## 13. 如何判断一个模块是否归档完成？

至少满足三个条件：关键材料齐全，说明文档清楚，运行或检查路径明确。完成后应更新 `00_admin/archive_status_table.md`。

## 14. 受版权限制的论文或数据可以提交吗？

应先确认版权和授权。不能公开提交的材料可提交引用说明、访问路径和许可证信息，不应直接上传受限内容。
