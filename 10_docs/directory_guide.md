# 目录使用指南

本文件解释本归档项目中各一级目录的用途、检查重点和交接方式。检查归档时，应同时查看目录内容和相关说明文档。

## `00_admin/`

用途：保存归档管理材料，包括检查清单、提交要求、缺失项、状态表、版本日志和学生通知。

检查时看什么：检查 `missing_items_todo.md` 是否更新，`archive_status_table.md` 是否反映实际提交情况，版本日志是否记录关键变更。

交接时怎么用：作为接收人了解当前归档进度和剩余工作的入口。

## `01_thesis/`

用途：保存论文 PDF、源文件、摘要、图表、评审意见、答辩版本和最终版本。

检查时看什么：论文版本是否完整，源文件是否能打开，图表是否与论文一致，最终版本是否明确。

交接时怎么用：用于确认论文成果和毕业归档版本。

## `02_papers/`

用途：保存与论文相关的投稿、发表、回复和手稿记录。

检查时看什么：投稿版本、发表版本、回复材料和手稿说明是否齐全。

交接时怎么用：用于了解论文成果转化和投稿状态。

## `03_methods/`

用途：保存第 3 至第 6 章方法代码、数据、配置、脚本、结果、图表和文档。

检查时看什么：每章是否有可运行代码、配置样例、运行脚本、原始输出和 README 说明。

交接时怎么用：后续学生可从章节 README 进入，定位方法代码和复现实验。

## `04_system/`

用途：保存第 7 章系统实现，包括前端、后端、部署、数据库、接口文档、截图、视频、用户手册和测试记录。

检查时看什么：系统是否能启动，部署说明是否清楚，截图和视频是否覆盖核心流程。

交接时怎么用：用于系统演示、后续维护和功能扩展。

## `05_datasets/`

用途：保存数据集原始数据、处理数据、样例、许可证、元数据和数据说明。

检查时看什么：数据来源、许可证、是否可公开、预处理方式和样例是否清楚。

交接时怎么用：用于复现实验前的数据准备和合规检查。

## `06_experiments/`

用途：保存按数据集和实验类型组织的实验材料。

检查时看什么：每个实验是否包含配置、命令、日志、原始输出、统计脚本和结果说明。

交接时怎么用：用于复现论文主要实验和定位结果来源。

## `07_results/`

用途：保存最终结果材料，包括汇总表、原始输出、最终图表和案例分析。

检查时看什么：论文中的表格和图是否能追溯到源数据和统计脚本。

交接时怎么用：用于快速复查论文结论和准备后续论文扩展。

## `08_ppt_and_reports/`

用途：保存答辩 PPT、组会 PPT、开题报告、中期报告、结题报告、项目申请和其他报告。

检查时看什么：重要阶段报告是否完整，PPT 中图表是否与论文结果一致。

交接时怎么用：用于了解项目过程和汇报历史。

## `09_review_and_graduation_materials/`

用途：保存评审意见、修改说明、答辩材料、毕业表格、签字扫描件、归档扫描件和正式文件。

检查时看什么：正式文件是否清晰完整，评审意见与修改说明是否逐条对应。

交接时怎么用：用于毕业流程核查和学院归档。

## `10_docs/`

用途：保存项目说明、复现指南、环境配置、目录指南、科研制品说明、归档规则、FAQ 和交接指南。

检查时看什么：说明是否与实际材料一致，复现步骤是否可执行。

交接时怎么用：作为新接收人理解项目的主入口。

## `11_weekly_and_process_records/`

用途：保存周报、会议纪要、任务记录、进度快照和项目时间线。

检查时看什么：关键节点是否记录，实验和论文推进是否有过程依据。

交接时怎么用：用于理解项目决策、阶段计划和问题演化。

## `tools/`

用途：保存归档辅助脚本。

检查时看什么：`generate_tree.py` 是否能生成 `tree.txt`，`check_missing_items.py` 是否能输出缺失项报告。

交接时怎么用：每次新增材料后运行脚本，保持目录树和缺失项报告更新。
