# 第 3 章：基于跨模型协同的代码生成方法

## 章节主题

本目录归档论文第 3 章“基于跨模型协同的代码生成方法”相关材料。该章节面向函数级代码生成任务，利用不同代码大模型在语义理解、边界条件处理、代码风格和测试通过率方面的差异，构建跨模型候选生成、互评、筛选与融合机制。

## 研究问题

1. 单一模型在复杂代码生成任务中容易出现语义偏差、边界遗漏和不可执行代码。
2. 不同模型生成结果之间存在互补性，但需要有效机制进行候选选择和质量判断。
3. 代码生成结果不仅要语法正确，还要通过测试并满足题目语义约束。
4. 跨模型协同应能在质量提升和调用成本之间取得可解释的平衡。

## 方法组成

- 多模型候选生成：调用多个代码模型或多种提示策略生成候选代码。
- 候选规范化：统一代码格式、函数签名、依赖导入和输出结构。
- 交叉评估：利用模型互评、规则检查、测试执行或评分器评估候选质量。
- 协同选择：结合测试结果、静态检查、模型置信度和历史表现选择最终答案。
- 失败样例分析：记录候选失败原因，为后续修复和提示优化提供依据。

## 关键输入输出

| 类型 | 内容 | 说明 |
|---|---|---|
| 输入 | 编程题描述、函数签名、样例测试、隐藏测试或公开测试 | 应保留原始任务文件和数据来源 |
| 输入 | 模型配置、提示词模板、解码参数 | 放入 `configs/` 或 `../shared_resources/prompts/` |
| 输出 | 多模型候选代码 | 放入 `results/` 或实验目录原始输出 |
| 输出 | 候选评分、测试结果、选择记录 | 应保留可复查的 JSON、CSV 或日志 |
| 输出 | 最终提交代码与统计表 | 对应论文第 3 章实验结论 |

## 需归档的代码和数据

- `code/`：跨模型候选生成、候选规范化、评估和选择逻辑。
- `data/`：第 3 章使用的数据子集、任务列表和样例输入输出。
- `configs/`：模型名称、温度、采样数量、评测参数和数据路径配置。
- `scripts/`：批量运行、结果汇总、失败案例提取和图表生成脚本。
- `docs/`：方法流程说明、模块接口说明、提示词设计说明。

## 实验材料

需要归档 HumanEval、MBPP、HumanEvalPlus 或其他函数级基准上的实验材料，包括：

1. 实验命令和运行环境。
2. 每个模型的候选生成数量和参数。
3. 每轮评估、互评、筛选和最终选择记录。
4. Pass@k、编译通过率、测试通过率、成本和耗时统计。
5. 与单模型、投票、简单重采样等基线的对比结果。
