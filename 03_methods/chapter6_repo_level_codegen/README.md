# 第 6 章：基于上下文感知与记忆增强的仓库级代码生成方法

## 章节主题

本目录归档论文第 6 章“基于上下文感知与记忆增强的仓库级代码生成方法”相关材料。该章节面向仓库级代码任务，强调跨文件上下文理解、依赖关系建模、结构化记忆维护和多轮补丁修复。

## 仓库上下文感知初始化

仓库级任务需要在生成前理解项目结构和任务位置。归档内容应包括：

- 仓库扫描和文件索引逻辑。
- 目标问题、相关文件、调用关系和依赖路径的识别方法。
- 上下文压缩、排序和截断策略。
- 初始化提示词、检索结果和上下文构建样例。

## 修复循环

修复循环用于在仓库级任务中持续生成、应用、测试和调整补丁。每轮应保留：

1. 当前仓库状态和补丁版本。
2. 被选择的上下文文件和原因。
3. 生成的补丁和修改说明。
4. 测试命令、测试日志和失败信息。
5. 下一轮上下文调整或停止条件。

## 结构化记忆

结构化记忆用于保存任务信息、文件关系、历史补丁、失败原因和验证结果。归档时需说明：

- 记忆的数据结构和字段含义。
- 记忆写入、更新、检索和淘汰规则。
- 记忆在上下文选择和补丁生成中的作用。
- 有无记忆增强的消融实验结果。

## RepoExec 与 EvoCodeBench 实验归档要求

第 6 章对应的仓库级实验数据包括 RepoExec 和 EvoCodeBench。两类数据集均需按独立目录归档，分别保留任务来源、仓库版本、运行环境、上下文构建记录、补丁生成过程、测试日志、结果文件和统计说明。

### RepoExec 实验材料

请在 `06_experiments/RepoExec/` 中补充：

- 任务列表、仓库来源、commit 或版本信息。
- 环境构建脚本、依赖安装方式和测试命令。
- 每个任务的上下文初始化结果。
- 每轮补丁、测试日志、最终状态和失败原因。
- 结果文件、统计脚本和与论文表格对应的说明。

### EvoCodeBench 实验材料

请在 `06_experiments/EvoCodeBench/` 中补充：

- EvoCodeBench 的数据来源、版本、任务列表和仓库快照信息。
- 任务执行环境、依赖安装方式、测试入口和评价脚本。
- 上下文感知初始化结果、结构化记忆记录和多轮修复日志。
- 最终补丁、测试结果、失败原因和案例分析。
- 结果文件、统计脚本和与 RepoExec 对照分析的说明。

### 统计说明

RepoExec 和 EvoCodeBench 的统计口径应分别记录，并在汇总结果中明确区分。建议至少保留任务通过率、补丁正确性、平均修复轮次、上下文长度、记忆命中情况、运行成本和时间消耗。若论文中合并展示两类数据集结果，应同时保存分数据集结果和合并统计脚本。

如本章还使用其他仓库级或类级数据集，应在 `05_datasets/dataset_description.md` 和 `06_experiments/` 中补充相同级别的说明。
