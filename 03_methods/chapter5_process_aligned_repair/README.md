# 第 5 章：基于过程对齐的测试驱动代码生成与修复方法

## 章节主题

本目录归档论文第 5 章“基于过程对齐的测试驱动代码生成与修复方法”相关材料。该章节关注代码生成过程与测试反馈过程之间的一致性，通过测试驱动、过程对齐、仲裁分析和最优解选择，使生成结果逐步接近任务需求。

## 初始代码与测试生成

方法首先根据题目描述、函数签名和约束生成初始代码，同时生成或扩展测试用例。归档时应保留：

- 初始代码生成提示词和模型配置。
- 测试生成提示词、测试用例格式和覆盖目标。
- 初始代码与初始测试的对应关系。
- 测试失败、断言错误和异常信息。

## 过程对齐循环

过程对齐循环用于比较“预期行为、测试反馈、代码修改意图和实际补丁”之间的关系。每轮循环应记录：

1. 当前代码版本。
2. 当前测试集合。
3. 失败测试和错误信息。
4. 对齐分析结论。
5. 生成补丁及其解释。
6. 测试结果变化和是否进入下一轮。

## 仲裁与对齐分析

仲裁或对齐分析模块负责判断多个候选代码或修复路径是否符合任务语义和测试意图。归档内容应包括：

- 仲裁提示词、评分规则或决策逻辑。
- 候选代码之间的差异比较。
- 测试覆盖不足、测试误导和过拟合风险判断。
- 仲裁输出样例和被采纳候选的理由。

## 最优解选择

最终答案选择不应只依据单次测试结果，还应综合语义一致性、测试通过情况、代码简洁性、边界条件覆盖和历史轮次表现。需要保留：

- 候选集合和候选来源。
- 每个候选的测试结果和分析评分。
- 最终选择规则。
- 被拒绝候选的主要原因。

## 需补交材料与说明

- `code/`：初始代码生成、测试生成、过程对齐循环、仲裁分析和最优解选择实现。
- `data/`：用于第 5 章实验的数据子集、测试用例、任务列表和失败样例。
- `configs/`：模型参数、循环轮数、测试生成参数、仲裁阈值和数据路径。
- `scripts/`：批量运行、测试执行、候选比较、结果汇总和图表生成脚本。
- `results/`：每轮代码、测试、对齐分析、仲裁输出、最终结果和失败案例。
- `figures/`：流程图、对齐循环示意图、结果对比图和消融分析图。
- `docs/`：方法说明、模块接口、实验说明和与论文段落对应关系。
