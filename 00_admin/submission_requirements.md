# 学生提交要求说明

为保证硕士毕业归档材料完整、清晰、可检查、可交接，请学生按照本说明整理并提交所有材料。提交前请完成自查，并在 `00_admin/missing_items_todo.md` 中更新实际状态。

## 一、命名规范

1. 文件名应包含材料类型、章节或实验名称、版本和日期，例如 `chapter3_cross_model_codegen_results_v1_20260510.xlsx`。
2. 论文、报告和 PPT 应保留最终版本标识，例如 `thesis_final_yewenguang_2026.pdf`。
3. 代码压缩包或外部仓库镜像应注明来源和版本，不建议使用 `final`、`new`、`latest` 等无法追溯的名称。
4. 文件名避免空格、特殊符号和中文标点，推荐使用小写英文、数字、下划线和短横线。

## 二、目录规范

1. 材料必须放入对应目录，不得集中堆放在根目录。
2. 每个章节方法的代码、数据、配置、脚本、结果、图表和说明应分别放入对应子目录。
3. 共享代码、提示词和评测工具应放入 `03_methods/shared_resources/`，不要重复复制到多个章节目录。
4. 若某项材料因版权、保密或体积原因无法提交，应在对应目录中放置说明文件，写明原因、存储位置和联系人。

## 三、论文材料要求

1. 必须提交论文最终 PDF、论文源文件、答辩版本和最终归档版本。
2. 图、表、算法、公式和实验结果应能在论文源文件中定位到对应源材料。
3. 论文评审后如发生修改，需提交修改说明或版本差异说明。
4. 摘要、关键词和中英文题目应与最终论文保持一致。

## 四、代码材料要求

1. 提交完整可运行代码，不得只提交关键片段。
2. 每个方法目录应提供依赖说明、运行入口、配置样例和预期输出位置。
3. 涉及模型接口、密钥或私有服务地址时，必须使用示例配置，不得提交真实密钥。
4. 若代码依赖外部仓库，应说明 commit、分支、许可证和本项目修改内容。

## 五、数据材料要求

1. 数据需说明来源、下载时间、许可证、是否可公开和使用限制。
2. 数据量较大时，可提交样例、元数据、下载脚本和校验值，并说明完整数据存放位置。
3. 预处理数据需提供预处理脚本、参数和处理日志。
4. 涉及个人信息、账户信息、私有仓库或商业数据时，必须脱敏。

## 六、实验材料要求

1. 每组实验应包含实验配置、运行命令、日志、原始输出、结果统计和评价指标说明。
2. 对 HumanEval、MBPP、HumanEvalPlus、BigCodeBench、ClassEval、RepoExec 和 EvoCodeBench 的实验，应按数据集目录分别归档。
3. 消融实验需说明去除或替换的模块、对照组设置和结论。
4. 成本分析需提交 token、时间、调用次数、模型费用或机器资源统计依据。

## 七、图表和表格源文件要求

1. 最终论文中出现的图表必须提交源文件或生成脚本。
2. 图表文件应能对应到论文图号或表号。
3. 汇总表应保留原始统计数据，避免只提交截图。
4. 若使用人工整理数据，应说明数据来源和整理规则。

## 八、PPT 和报告要求

1. 提交答辩 PPT、开题报告、中期报告和重要组会汇报。
2. PPT 中引用的核心图表应与论文和结果目录中的文件保持一致。
3. 报告文件应注明提交阶段、版本和日期。
4. 如存在导师修改意见，可与报告一起归档。

## 九、评审意见与修改说明要求

1. 论文评审意见、答辩意见和修改说明必须完整保存。
2. 修改说明建议采用“意见编号、原意见、修改位置、修改内容、是否完成”的表格格式。
3. 对未采纳意见需说明理由。
4. 签字、盖章和正式表格扫描件应清晰可读。

## 十、脱敏与版权要求

1. 不得提交真实 API Key、访问令牌、密码、私有数据库连接串和个人隐私信息。
2. 第三方数据、代码、模型和图片需保留许可证或引用来源。
3. 受版权限制无法公开的材料，应提交说明文件而非直接上传。
4. 大文件应按导师要求使用外部存储，并在目录中记录校验值和访问方式。

## 十一、截止前自查要求

1. 运行 `python3 tools/generate_tree.py` 更新目录树。
2. 运行 `python3 tools/check_missing_items.py` 生成缺失项报告。
3. 更新 `00_admin/archive_status_table.md` 和 `00_admin/missing_items_todo.md`。
4. 检查每个新增文件是否命名清晰、位置正确、可打开。
5. 检查论文、代码、实验、数据、系统和毕业材料是否能相互对应。
