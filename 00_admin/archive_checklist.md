# 归档检查清单

本清单用于导师、学生和后续接收人核对归档完整性。勾选前应确认材料已经放入指定目录，并且文件命名、版本、来源和可复现说明清楚。

## 一、论文材料

- [ ] 必须提交：论文最终 PDF，放入 `01_thesis/thesis_pdf/`。
- [ ] 必须提交：论文源文件，放入 `01_thesis/thesis_source/`。
- [ ] 必须提交：中文摘要、英文摘要和关键词文件，放入 `01_thesis/abstract/`。
- [ ] 必须提交：答辩版本论文，放入 `01_thesis/defense_version/`。
- [ ] 必须提交：最终归档版本论文，放入 `01_thesis/final_version/`。
- [ ] 必须提交：论文修改前后差异说明，放入 `01_thesis/review_comments/` 或 `09_review_and_graduation_materials/revision_explanations/`。

## 二、代码材料

- [ ] 必须提交：第 3 章跨模型协同代码生成方法代码，放入 `03_methods/chapter3_cross_model_codegen/code/`。
- [ ] 必须提交：第 4 章轨迹驱动修复方法代码，放入 `03_methods/chapter4_trace_driven_repair/code/`。
- [ ] 必须提交：第 5 章过程对齐测试驱动修复方法代码，放入 `03_methods/chapter5_process_aligned_repair/code/`。
- [ ] 必须提交：第 6 章仓库级代码生成方法代码，放入 `03_methods/chapter6_repo_level_codegen/code/`。
- [ ] 必须提交：共享提示词、公共工具和评测组件，放入 `03_methods/shared_resources/`。
- [ ] 建议提交：核心算法伪代码和接口说明，放入各章节 `docs/`。

## 三、数据材料

- [ ] 必须提交：原始数据来源说明，放入 `05_datasets/raw/` 和 `05_datasets/dataset_description.md`。
- [ ] 必须提交：预处理后数据或生成方式，放入 `05_datasets/processed/`。
- [ ] 必须提交：数据样例，放入 `05_datasets/samples/`。
- [ ] 必须提交：许可证、使用限制和不可公开说明，放入 `05_datasets/licenses/`。
- [ ] 建议提交：数据字段元信息和统计摘要，放入 `05_datasets/metadata/`。

- [ ] 必须提交：HumanEvalPlus 实验配置、日志、输出和结果说明。
- [ ] 必须提交：BigCodeBench 实验配置、日志、输出和结果说明。
- [ ] 必须提交：ClassEval 实验配置、日志、输出和结果说明。
- [ ] 必须提交：RepoExec 实验配置、日志、输出和结果说明。
- [ ] 必须提交：EvoCodeBench 实验配置、日志、输出和结果说明。
