# 结果文件组织说明

本目录用于集中保存论文实验结果、统计表、最终图表和案例分析。结果材料应能从原始输出追溯到论文中的表格、图和结论。

## 一、原始输出

`raw_outputs/` 用于保存模型原始输出、程序运行日志、测试日志、智能体对话记录、补丁版本、轨迹文件和失败记录。原始输出应尽量保持生成时的结构，不建议只保留人工整理后的摘要。

建议命名示例：

- `humaneval_chapter3_cross_model_raw_20260513.jsonl`
- `repoexec_chapter6_patch_logs_task001_20260513.json`
- `evocodebench_chapter6_patch_logs_task001_20260513.json`
- `chapter4_trace_repair_round_logs_20260513/`

## 二、汇总表

`summary_tables/` 用于保存可直接支撑论文结果表的数据文件。建议使用 CSV、XLSX 或 Markdown 表格，并保留统计脚本路径。

汇总表应说明：

1. 数据来源。
2. 统计口径。
3. 指标定义。
4. 是否包含失败任务。
5. 与论文表号的对应关系。

## 三、最终图表

`final_figures/` 用于保存论文最终使用的图表文件和源文件。每个图建议同时保存：

- 可编辑源文件或绘图脚本。
- 导出的 PNG、PDF 或 SVG 文件。
- 图表对应的源数据。
- 与论文图号的对应说明。

## 四、案例分析

`case_studies/` 用于保存成功案例、失败案例、修复轮次分析、运行轨迹分析、仓库级补丁分析和系统演示案例。每个案例应包含任务背景、输入、过程、结果和结论。

## 五、复现要求

请学生提交可复现的统计脚本与源数据。只提交最终截图或手工整理结果无法满足归档要求。若结果依赖外部模型服务，应记录模型名称、接口版本、参数、调用时间和成本统计。
